# A3TAXI — Taxi Fleet Management Web App

Web app for managing a small taxi fleet (~15 vehicles/drivers) with two roles:

- **Admin**: client accounts, billing/invoicing, dispatching, reservations
- **Driver**: own dashboard — trip entry, receipt uploads, dispatched jobs, GPS sharing. Logs in via an access code generated by the admin.

## Modules

1. **Billing / Invoicing** — client accounts, driver trip entry with receipt photos, monthly invoice generation
2. **Dispatching** — real-time driver locations on a map, job dispatch to drivers
3. **Online Reservations** — public booking form, admin calendar, SMS confirmation

## Tech Stack

- **Frontend**: React
- **Backend**: Node.js / Express
- **Database**: PostgreSQL
- **Maps/Geolocation**: Google Maps API
- **SMS**: Twilio
- **Auth**: JWT-based admin login + driver access codes

## Project Structure

```
A3TAXI/

```

## Local Setup

### Prerequisites
- Node.js 18+
- PostgreSQL 14+

### Backend
```bash
cd backend
npm install
cp ../.env.example .env   # fill in values
npm run migrate           # create schema
npm run seed              # optional: dev sample data
node src/config/seedAdmin.js <email> <password> "<name>"   # create your first admin login
npm run dev
```

### Frontend
```bash
cd frontend
npm install
cp .env.example .env      # fill in VITE_GOOGLE_MAPS_API_KEY, etc.
npm run dev
```

### Environment Variables
See `.env.example` at the repo root for backend variables (database URL, JWT secret, Twilio credentials), and `frontend/.env.example` for frontend variables (API URL, Google Maps key).
