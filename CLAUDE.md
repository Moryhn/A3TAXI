# CLAUDE.md — Taxi Fleet Management Web Application

## General Context

Small taxi fleet managing approximately **15 vehicles / drivers**. The owner (admin) wants a web app with **two access roles**:

- **Admin**: full access (client accounts, billing, dispatching, reservations)
- **Driver**: limited access to their own dashboard, login via **access code generated by admin**

The app will be developed with Claude Code and the source code must be pushed to a **GitHub repository** (to be created or connected).

---

## Module 1 — Billing / Invoicing

### Client / Vendor Accounts
- Admin creates and manages **client accounts** (e.g., "DHL", "Company Y", etc.)
- Each client account has a unique ID selectable by drivers

### Trip Entry by Driver
For each trip, the driver must be able to enter:
- The **client account** (selection from a list pre-populated by admin)
- **Departure location**
- **Arrival location**
- **Trip amount**
- **Receipt photo** (proof), uploaded and **saved in the system**, viewable/searchable later by admin (by date, driver, or client account)

### Automatic Invoicing
- Each trip entered automatically accumulates under the corresponding client account
- At month-end, admin can **generate/print an invoice** per client, listing all associated trips (date, driver, route, amount, total)
- Admin sends the invoice themselves (no automatic sending required for now)
- Receipt photos are **not** included on the printed invoice — they remain only in the system for internal verification

---

## Module 2 — Dispatching

- Real-time map (**Google Maps API**) displaying the location of each active driver
- Admin must be able to see where each driver is at all times
- Admin can **send an address/job** to a specific driver (notification/dispatch)
- Driver receives the assigned job on their dashboard
- Requires: real-time geolocation on driver side (mobile), position storage, notification system (push or in-app)

---

## Module 3 — Online Reservations

- Clients can **book online** (public form)
- Reservation automatically added to **admin's calendar**
- Automatic **SMS confirmation** sent to client (Twilio-type integration)
- Calendar must reflect existing availability/reservations

---

## Roles and Access

| Role | Access |
|---|---|
| Admin | Manage client accounts, invoicing, full dispatch view, manage reservations/calendar, generate driver access codes |
| Driver | Own dashboard: trip entry, receipt photo upload, receive dispatched jobs, share GPS location |

- Driver login: via **unique access code** generated by admin (no self-signup)

---

## Suggested Tech Stack (to validate/adjust with user)

- **Frontend**: React (responsive web, usable on mobile by drivers)
- **Backend**: Node.js / Express or equivalent
- **Database**: PostgreSQL (relations: client accounts ↔ trips ↔ drivers ↔ invoices)
- **Photo Storage**: file storage service (e.g., AWS S3 or equivalent)
- **Maps/Geolocation**: Google Maps API (Maps JavaScript API + Geolocation)
- **SMS**: Twilio
- **Authentication**: access code system for drivers + classic admin login

---

## Suggested Development Priorities

1. Core structure: admin/driver auth, database (client accounts, drivers, trips)
2. Billing module (trip entry, photo upload, invoice generation)
3. Dispatching module (map, positions, job dispatch)
4. Reservations module (public form, calendar, SMS)

## GitHub Repository Setup Instructions for Claude Code

### Initial Setup

**If the user does not have an existing GitHub repository:**
1. Create a new directory locally with the project name: `taxi-fleet-app` (or user-specified name)
2. Initialize Git: `git init`
3. Create a `.gitignore` file (include: `node_modules/`, `.env`, `.env.local`, `dist/`, `build/`)
4. Create an initial `README.md` with project description and setup instructions
5. Make an initial commit: `git add . && git commit -m "Initial commit: project structure"`
6. **Ask the user for their GitHub username and create instructions** for pushing to a new repo (or offer to guide them through repo creation on GitHub)

### If user provides an existing GitHub URL:
1. Clone or connect to the existing repository
2. Pull the latest changes: `git pull origin main`
3. Create a feature branch for this app: `git checkout -b feature/taxi-app`

### During Development
- Use meaningful commit messages: `git commit -m "feat: add driver login with access codes"`
- Push regularly: `git push origin [branch-name]`
- When features are complete, create Pull Requests (if using a team workflow) or push to `main`
